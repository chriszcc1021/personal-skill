# voiceprint MVP 验收标准

> 没有尺子,做完了不知道成没成。定清楚"到什么程度算过关"。

## 1. 技术验收
- [ ] 上传任意格式音频,后端能统一处理不报错
- [ ] 单次匹配端到端 < 3 秒
- [ ] 6 种错误码都能正确触发并友好提示
- [ ] 同一个人录两次,Top 匹配结果基本一致(稳定性)
- [ ] 50 并发不崩

## 2. 效果验收(最关键,靠探针数据定)
- [ ] **区分度**:不同用户测出来的 Top 匹配不能全是同一个歌手(不然模型没在工作)
- [ ] **合理性**:找 5-10 个真人测,让他们自评"像不像",主观认可率 ≥ 50% 算及格,≥ 70% 算好
- [ ] **稳定性**:同人同段音频多次测,Top1 不变

## 3. 体验验收
- [ ] 手机 H5 录音流畅,不卡
- [ ] 结果页一眼看懂
- [ ] 有分享欲(测试者愿意截图发出去)

## 4. 红线(不过关就不上线)
- 结果全同一个歌手 → 模型/库有问题
- 主观认可率 < 30% → 方案要重估(可能转音域匹配)
- 延迟 > 8 秒 → 体验崩

## 5. 分阶段 Gate
1. **探针 Gate**:3-5 歌手小库,验证唱歌能否匹配 + 区分度 → 过了才建大库
2. **MVP Gate**:26 歌手库 + Web 前端,达上述验收 → 过了才考虑小程序
3. **小程序 Gate**:复用后端,前端跑通 + 过审
