# voiceprint 相似度映射方案

> 问题:余弦相似度算出来是 0.x 的原始值,直接给用户没意义还打击体验。
> 目标:映射成"看着合理、有区分度、不离谱"的百分比。

## 1. 原始相似度的现实
- speaker embedding 余弦相似度:同一个人多段通常 0.7-0.9,不同人 0.2-0.5
- 用户 vs 歌手大概率落在 0.3-0.6,直接显示"你像周杰伦 45%" → 用户觉得"这么低?不准"

## 2. 映射策略(三选一,建议 C)

### A. 线性拉伸
把 [0.3, 0.7] 线性映射到 [60%, 95%]。简单,但容易失真。

### B. 相对排名制
不显示绝对分,只显示"最像的 Top3 + 相对强弱条"。规避绝对值尴尬,但用户想看具体数字。

### C. 混合制(推荐)
- 内部保留真实余弦值用于排序
- 展示:Top1 映射到 80-92% 区间,Top2/3 按差距递减
- 加"匹配维度"文字("音色低沉度接近""气声比例相似")增强可信,弱化对数字的纠结

## 3. 防离谱兜底
- 若 Top1 原始相似度过低(如 < 0.35)→ 提示"你的声线很独特,没有特别接近的歌手" 或 降级展示,别硬凑一个高分
- 若多个歌手分数挤在一起 → 都展示,标"你介于XX和XX之间"

## 4. 调参说明
- 映射区间的具体阈值,**必须拿真实测试数据调**(探针阶段产出)
- 先粗设,MVP 上线后看用户反馈迭代
