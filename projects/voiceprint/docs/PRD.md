# voiceprint PRD — 产品需求文档 (MVP)

## 1. 一句话定义
用户录一段自己的声音,测出音色最像哪些歌手,给出相似度分数,可分享。

## 2. 目标用户
KTV 爱好者、爱玩声音测试的年轻人、想知道自己"声线像谁"的普通用户。

## 3. 用户旅程 (MVP)
1. 打开网页 → 看到"测测你的歌声像哪个歌手"引导
2. **选歌词 + 试听（解决“不知道唱啥”）**:
   - 页面给几首推荐曲目(副歌辨识度高的句子)
   - 每句歌词旁边 ▶️ 播放钮,点了听一小段(**B方案:短片段几秒,合理使用擦边**)
   - 用户听完选中某句 → 歌词高亮
3. 点麦克风 **跟着清唱**这句(15-30s,必须唱不是说,安静环境)
4. 录完 → 上传 → 等待(转圈,几秒)
5. 出结果(双维度):
   - **音色维度**:Top3 匹配歌手 + 相似度百分比
   - **音域维度**:你的音域/声部 + 音域最接近的歌手
   - 一张可分享的结果卡片
6. 可重录 / 可分享

> ⚠️ 同源红利:推荐曲目固定 → 歌手库用同一片段建库 → 用户唱的句 ↔ 歌手唱的同句,同源同段比对,准度拉满。
> ⚠️ B方案版权:试听用短片段擦合理使用,规模化后若有风险再评估转伴奏/清唱示范或买授权。

## 4. 功能范围
### MVP 必做 (P0)
- [ ] **推荐曲目 + 歌词试听**(B方案短片段,解决“不知道唱啥”)
- [ ] 浏览器录音(15-30s,时长校验,引导清唱)
- [ ] 音频上传后端
- [ ] 声纹提取 + 歌手库比对
- [ ] **音域/声部检测(第二维度,张鱼哥定为P0)**
- [ ] 返回双维度结果(音色 tone_matches + 音域 range_matches)
- [ ] 结果页展示
- [ ] 基础错误处理(录音太短/太吵/无人声)

### V1.1 可做 (P1)
- [ ] 结果分享卡片(生成图片,Spotify Wrapped 风)
- [ ] 歌手库扩到 100+

### 暂不做
- 用户账号系统
- 历史记录
- 付费功能
- 微信小程序(Phase 2)

## 5. 非功能需求
- 单次匹配延迟 < 3 秒
- 支持并发(先扛 50 并发,后续加机器)
- 移动端 H5 优先(大部分人手机玩)

## 6. 结果呈现原则
- 定位娱乐,不号称精准
- 分数别给太低(打击体验),做映射拉到"看着合理"区间
- 每个结果配"你和XX共同点:音色偏XX"这种解释,增强可信+趣味
