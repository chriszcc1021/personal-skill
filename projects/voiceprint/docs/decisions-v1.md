## §调整:相比原 MVP 文档,需要改动的内容

### 调整1:歌手库 26 → 50,且四大区域配比要重定
原 singer-library 名单是 26 人草案。现在扩到 50,需重新分配:
- 内地 CN:~12
- 港澳台 TW/HK:~12
- 日本 JP:~10
- 欧美 US/UK:~12
- 韩国 KR:原来有,张鱼哥这次没提 → **待确认:50 里含不含韩国?** 若含,各区压缩;若不含,按上面四区分。
→ 影响:建库工期从 1.5-2 天 → **约 3-4 天**(50 人)

### 调整2:呈现风格锁定 Spotify Design
原 PRD 说"科学报告感 vs 娱乐爆款感"二选一 → 现定 **Spotify 风**:
- 深色背景 (#121212) + Spotify 绿 (#1DB954)
- 结果页做成"歌手卡片"横滑,大头像 + 匹配度进度条
- 分享卡片模仿 Spotify Wrapped 年度总结那种视觉
→ 影响:前端设计有明确参照物,好做;需补一份 UI 设计规范

### 调整3:音域检测从 P1 提到 P0
原 mvp-acceptance 把音域放 V1.1。现在张鱼哥要求 MVP 就带音域 → 提到 P0:
- 需集成音高检测(pitch detection,如 CREPE / pyin / librosa)
- 结果页多一个模块:你的音域 F2-G4 + 声部(男中音)+ "和XX歌手音域接近"
- 音域匹配比声纹匹配更硬核可靠 → 可作为"可信度锚点",弥补声纹玄学
→ 影响:后端多一个 pitch 分析管线,api-spec 的 vocal_range/voice_type 字段从可选变必填

### 调整4:结果 = 双维度呈现
现在结果有两条线:
- **声纹音色维度**:你音色像谁(玄学但好玩)
- **音域维度**:你音域像谁 + 声部(硬核可信)
→ 呈现上要把两者组合,api-spec 的 matches 可能要分 tone_matches / range_matches 两组

### 调整5:素材源现实问题(最关键,见 material-source.md)
"用现成 + 我找素材"有个硬现实:研究数据集里的干声都是**匿名歌手**,没有周杰伦/VAUNDY 这种指名歌手的干声。指名歌手只能走"歌曲→Demucs 分离"。这条要张鱼哥知晓并接受。

## 张鱼哥二次确认 (2026-07-19 01:28)
1. **不含韩国** → 50 人四区分配:内地~13 / 港台~13 / 日本~12 / 欧美~12
2. **访谈路线砍掉** → 张鱼哥关键纠正:「访谈声音不是唱歌声音」。完全正确 —— 唱歌的音高/共鸣/发声方式跟说话差很远,拿说话干声建库去匹配用户唱歌,等于拿苹果比橘子。
   → **建库素材必须是唱歌干声**,不能用访谈说话。
   → 主力路线定为:歌曲 + Demucs 分离唱歌人声(+ official acapella 优先)。访谈说话不再作为素材源。
   → 连带影响:用户端也应引导「唱一段」而非「说一段」,保证输入输出同源(都是唱歌)。
