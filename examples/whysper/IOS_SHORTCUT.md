# Whysper iOS 快捷指令配置

## 目标
按 Action Button（长按）→ 截屏 → 自动上传到 Whysper → AI 自动判断去向：

- 消费/账单截图 → 进入「账」的待确认队列
- 有时间的事项 → 生成日历 / Todo 提醒时间项
- 普通截图 / 文章 / 笔记 → 进入知识整理

## 配置步骤

### 1. 在 Whysper 里复制接口地址

打开 Whysper 记录页，点「Action Button」配置提示里的「配置」。

页面会显示当前部署环境对应的上传地址，通常形如：

`https://你的域名/whysper-api/entries`

点「复制 URL」，再打开「快捷指令」App。

> 不要固定使用旧的临时隧道域名。隧道重启后域名会变，以 Whysper 页面里显示的 URL 为准。

### 2. 打开「快捷指令」App，新建快捷指令

### 3. 添加动作（顺序）

| # | 动作 | 参数 |
|---|---|---|
| 1 | **截屏** | （iOS 自动捕获当前屏幕，分享时会跳出隐私确认） |
| 2 | **获取「截屏」的内容** | 用上一步输出 |
| 3 | **通过 URL 获取内容** | URL: 使用 Whysper 页面里复制的地址 |
|   |   | 方法: **POST** |
|   |   | 请求体: **表单** |
|   |   | 字段 1: `image` (文件) = 上一步输出 |
|   |   | 字段 2: `source` (文本) = `shortcut` |
|   |   | 字段 3: `capture_mode` (文本) = `auto`，可省略 |
| 4 | **获取词典值** | 从上一步响应里取 `id` |
| 5 | **等待** | 5 秒 |
| 6 | **通过 URL 获取内容** | URL: `复制的地址` 去掉 `/entries` 后拼上 `/entries/{id}/calendar-items` |
|   |   | 方法: **GET** |
| 7 | **如果** | `has_items` 是 true |
| 8 | **重复处理每一项** | 读取 `items` |
| 9 | **添加新日程** | 标题 = `title`，开始 = `start_iso`，结束 = `end_iso`，备注 = `notes`，位置 = `location` |
| 10 | **打开 URL** | `https://你的域名/whysper/`，想直接看账本可打开 `https://你的域名/whysper/?tab=ledger` |
| 11 | **显示通知** | 有时间项: "已上传并加入日历"；无时间项: "已上传，AI 已处理" |

### 4. 命名 + 绑定 Action Button

1. 起名「Whysper 截图」
2. 设置 → Action Button → 选「快捷指令」→ 选「Whysper 截图」
3. 长按 Action Button 触发
4. 回到 Whysper，点「我已配置」

### 5. 首次运行
- 系统会弹「允许访问当前域名」→ 选「允许」
- 之后每次自动跑

## 注意

- **iOS 限制**：网页不能静默创建快捷指令，也不能自动绑定 Action Button。Whysper 只能复制接口地址、打开快捷指令 App，并提示你完成一次配置。
- **trycloudflare URL 不稳定**：临时隧道偶尔换域名，等做了固定域名再固化
- **响应快慢**：上传约 1-2 秒，AI 处理 5-15 秒（异步）。如果第 6 步还没拿到时间项，可以再加一次「等待 5 秒 → 获取内容」重试。
- **时间项来源**：Whysper 会把截图里的日程、待办截止时间、码过期时间统一返回为 `calendar-items`，快捷指令只需要按 items 添加日程。
- **账单来源**：如果 AI 判断截图是消费/账单，会进入「账」页待确认。不会自动入账，需要你确认金额和商家。
- **失败排查**：通知里加一行「获取上一步输出的 `状态`」就能看到 HTTP 状态码
