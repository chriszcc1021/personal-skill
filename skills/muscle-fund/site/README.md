# 肌肉基金 Muscle Fund 💪

> 不练就罚钱，越练越壮。基金里的钱攒着发奖。

## 在线地址

https://broadway-aim-discuss-occasions.trycloudflare.com/fastpublish/muscle-fund/

## 怎么用

### 1. 加成员
顶部导航 → **加成员** → 起名字 → 捏形象（性别 / 发型 / 眼镜 / 鼻子 / 嘴 / 短裤）→ 入队

### 2. 打卡
**打卡** 页 → 选成员 → 选今天练的部位（多选）→ 确认。一天只能打一次。

支持 6 个部位：
- 💪 手臂 / 🫁 胸肌 / 🦴 背 / 🧱 腹肌 / 🪨 肩 / 🦵 腿

每个部位打卡次数累计：
- 0-4 次：原始
- 5-9 次：开始有形
- 10-19 次：明显
- 20-29 次：壮
- 30+ 次：怪兽 🦾

各部位独立生长，不会"全身一起变大"。

### 3. 账本
**账本** 页：每人累计打卡天数、累计欠款、本月欠款。底部可导出 CSV。

### 4. 改罚款 / 管成员
**设置** 页：
- 改每次罚款金额（默认 ¥30）
- 删成员
- 导入 / 导出 JSON 数据备份
- 清空数据

## 结算逻辑

进任何页面都会自动跑一次 `settleOverdue()`：从你上次结算日期到昨天，遍历每一个**工作日（周一到周五）**，凡是没打卡的天数都会：
- 给该成员加一次 `fineAmount` 欠款
- 给基金加一次 `fineAmount`

周末不算。当天不算（怕还没到下班）。

## 数据存哪

全部在浏览器 `localStorage`（key=`mf_state_v1`）。**不在服务器**。换浏览器、清缓存就没了。  
重要数据请定期到 设置 → 导出 JSON 备份。

## 已知问题 / TODO

- 数据只在本地浏览器，没有云同步。要多人共享得手动导入导出 JSON。
- "本月欠款"是估算（实时算的，不含今天）；"累计欠款"才是真扣过的。
- 没有撤销打卡功能；不小心打错了只能 设置→清空 重来。
- 没有移动端深度优化（基本响应式 OK，但小屏体验可以更好）。
- 角色"背越练越宽"是把整个躯干拉宽，不是真后背肌肉。粗略实现。
