方案 3 — Rive 运行时演示

🚧 诚实说明 必读

没有 Rive 编辑器环境（一个 GUI 桌面/网页 app），没办法原创一只海绵宝宝鱼的 .riv 文件。 Rive 文件是二进制的，必须用 rive.app/editor 拖关节、画矢量、绑骨骼、设 state machine 才能导出。

所以方案 3 给两个东西：
1️⃣ 用 @rive-app/canvas 加载 Rive 社区的公开 .riv 文件演示 运行时能力（关节驱动、state machine、输入响应）。
2️⃣ 列出制作流程：如果哥要原创鱼，下面这套就是制作步骤。

🎬 Demo 1 · Rive 社区公开动画

加载 Rive CDN 上的 vehicles.riv（官方示例）。Rive 运行时全功能：矢量缩放无损、关节插值、文件 ~30 KB。

🛠️ 如果原创海绵宝宝鱼，制作步骤

1. 打开 rive.app/editor，新建 Artboard 400×400。
2. 矢量画鱼：椭圆身体（青蓝 #4FC3F7）+ 三角鳃 + 三角头鳍 + 大眼 + 厚黄唇 + 三角粉鼻。
3. 加 Bone：肩骨（2 根）→ 上臂骨 → 前臂骨 → 手骨，绑哑铃 group 到手骨末端。
4. 时间轴关键帧：0s 上臂 0° / 前臂 0° → 0.5s 上臂 -10° / 前臂 -140° → 1.0s 回 0°。
5. 加 State Machine：idle ↔ curl，input isLifting 触发切换。
6. 加配料动画：呼吸（身体 scale）、眨眼（眼睑 mask）、尾鳍摆（rotation）。
7. Export → spongebob-fish-curl.riv（~25-40 KB）。
8. 网页用 new rive.Rive({src:'fish.riv', artboard:'Fish', stateMachines:'sm'}) 加载。
Rive 优势：矢量无损 · 文件小（~30 KB vs PNG 序列 6 MB）· State Machine 交互 · 运行时控制参数
劣势：必须用 Rive 编辑器（无 code-only 工作流）· 学习成本高 · 社区资源比 Lottie 少
